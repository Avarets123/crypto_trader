Ты — эксперт в разработке алгоритмических торговых стратегий на криптобиржах. Твоя задача: написать код для реализации стратегии **микроскальпинга**, основанной на данных реального времени: стакан (order book), лента сделок (time & sales) и агрегированная статистика.

## 1. Входные данные (потоковое обновление)

Бот получает следующие данные в реальном времени через WebSocket:

- **Order Book (Стакан):** частичные обновления (depth 20-50 уровней). Структура: `bids = [[price, volume], ...]`, `asks = [[price, volume], ...]`.
- **Trades (Лента сделок):** каждая сделка содержит поля: `price`, `volume`, `side` (buy или sell, где buy = taker buy, sell = taker sell), `timestamp`.

Дополнительно бот хранит историю для расчёта скользящих метрик.

## 2. Расчетные метрики (обновляются на каждом событии)

### 2.1. Дисбаланс стакана с глубиной 1% (OBI_1pct)
- Определить текущую цену `mid_price = (bids[0].price + asks[0].price) / 2`.
- Вычислить границы: `lower = mid_price * 0.99`, `upper = mid_price * 1.01`.
- Просуммировать все объёмы `bids` в диапазоне [lower, mid_price] → `bid_vol`.
- Просуммировать все объёмы `asks` в диапазоне [mid_price, upper] → `ask_vol`.
- Рассчитать: `OBI = (bid_vol - ask_vol) / (bid_vol + ask_vol)`. Если обе суммы 0, `OBI = 0`.

### 2.2. Кумулятивная дельта объёма за 5 секунд (CVD_5s)
- Хранить кольцевой буфер сделок за последние 5 секунд.
- Для каждой сделки: если `side == 'buy'`, добавить `+volume`; если `'sell'`, добавить `-volume`.
- `CVD_5s = сумма значений в буфере`.

### 2.3. Скользящие статистики CVD_5s (за 1 час)
- Хранить значения `CVD_5s` за последний час (например, каждые 100 мс или по факту новой сделки).
- Вычислить:
  - `avg_CVD_1h = среднее арифметическое массива`
  - `std_CVD_1h = стандартное отклонение массива`
- Порог аномалии: `cvd_threshold = avg_CVD_1h + 2 * std_CVD_1h`.

### 2.4. Средний объём сделки за минуту (AvgTradeVol_1min)
- Хранить буфер объёмов всех сделок за последние 60 секунд.
- `AvgTradeVol_1min = среднее арифметическое буфера`.
- Определение «китовой сделки»: `is_whale = (trade.volume >= 3 * AvgTradeVol_1min) AND (trade.volume >= 0.5)`. (Для альткоинов можно снизить абсолютный порог).

### 2.5. Поиск крупной стены сопротивления (Resistance Wall)
- Сканировать стакан `asks` в диапазоне от текущей цены до `mid_price * 1.01` (1% вверх).
- Порог стены: `wall_threshold = 5 * AvgTradeVol_1min` (или, если для BTC, можно задать фиксированно 10 BTC).
- Найти **первый** уровень `asks[i]`, где `asks[i].volume >= wall_threshold`.
- Если такой найден, запомнить его цену `resistance_wall_price`.

## 3. Условия входа в позицию LONG (покупка)

Стратегия ищет момент, когда **рыночный ордер съедает всю ликвидность на первом уровне Ask**, после чего образуется дисбаланс в пользу покупателей и аномальный всплеск CVD.

**Проверять условия при каждом обновлении стакана или появлении новой сделки. Все условия должны выполниться одновременно в пределах окна ~100 мс.**

1. **Триггер «Съедена стена Ask1»**  
   - Последняя сделка является **taker buy** (`side == 'buy'`).  
   - Объём этой сделки `>= 0.9 * asks[0].volume` (или строго `>= asks[0].volume`).  
   - Сделка удовлетворяет условию `is_whale == True`.

2. **Дисбаланс стакана**  
   - Текущий `OBI_1pct > 0.6`.

3. **Всплеск агрессивного давления**  
   - Текущий `CVD_5s > cvd_threshold` (превышает avg + 2*std за час).

4. **Фильтр спреда**  
   - `(asks[0].price - bids[0].price) / bids[0].price < 0.0005` (спред менее 0.05%).

**Если все 4 условия истинны:**
- **Вход:** Рыночный ордер на покупку (market buy).
- **Объём позиции:** `min( 0.5 * asks[0].volume, максимально_допустимый_риск_в_базовой_валюте )`.  
- **Цена входа:** фактическая цена исполнения рыночного ордера (может быть чуть выше asks[0].price).

## 4. Управление позицией и выход

Сразу после исполнения входа необходимо установить защитные ордера (желательно одним OCO-запросом к бирже).

### 4.1. Тейк-профит (TP)
- Использовать `resistance_wall_price` (рассчитанную в п.2.5).  
- Если стена найдена: `TP_price = resistance_wall_price - 0.01` (минус 1 тик, если позволяет точность пары).  
- Если стена не найдена (в пределах 1%): `TP_price = entry_price * 1.002` (цель 0.2% прибыли).

### 4.2. Стоп-лосс (SL)
- `SL_price = entry_price * 0.9965` (жёсткий стоп -0.35% от входа).

### 4.3. Динамический выход по ослаблению CVD
Пока позиция открыта, бот **каждые 200-500 мс** проверяет:
- Пересчитать `CVD_5s` актуальный.
- Если `CVD_5s <= avg_CVD_1h` ИЛИ знак `CVD_5s` стал отрицательным:  
  **Немедленно закрыть позицию рыночным ордером на продажу** (отменив ранее выставленные TP/SL).

## 5. Важные замечания для реализации

- **Скорость:** Вся логика должна быть событийно-ориентированной (event-driven). Никаких `time.sleep()`. Использовать асинхронность (asyncio) или колбэки WebSocket.
- **Управление ордерами:** При входе рекомендуется выставлять OCO (One-Cancels-Other) с лимитным TP и стоп-маркет SL, чтобы минимизировать риск потери связи.
- **Комиссии:** Стратегия рассчитана на комиссию тейкера **не более 0.05%**. При большей комиссии порог `OBI` и `TP` нужно увеличить.
- **Спуфинг-защита:** Проверка `is_whale` по реальной сделке (а не только по наличию стены в стакане) частично защищает от ложных пробоев.
